1. Abstract

The IQRA Academy Student Management System (IASMS) is a comprehensive, client-side web application developed to digitize and streamline the administrative operations of IQRA Academy. Built entirely with frontend technologies — HTML5, CSS3, and Vanilla JavaScript — without any backend server or database, the system leverages the browser's native LocalStorage API for complete data persistence.

The system encompasses a full-featured ERP experience including secure login authentication, dashboard analytics with real-time charts, student enrollment and management (CRUD), date-wise attendance tracking, monthly fee management, and system settings with dark mode support. The interface follows modern UI/UX principles including glassmorphism, responsive design, and professional blue theming — making it suitable for university project submission, portfolio showcase, and internship demonstration.

2. Introduction

In the digital age, educational institutions require efficient, reliable, and user-friendly systems to manage their day-to-day administrative tasks. IQRA Academy, a forward-thinking educational institution operating under the motto "Learn • Grow • Succeed," recognized the need for a centralized platform to manage student records, track attendance, and handle fee collection without the complexity and cost of a full-stack application.

This project presents a modern, purely frontend solution that addresses all core administrative needs of the academy. By utilizing the powerful capabilities of modern browsers — including LocalStorage, CSS custom properties, and ES6+ JavaScript — the system provides a fully functional ERP interface that requires zero installation, zero server configuration, and zero database setup. It runs directly in any modern web browser.

The system was designed with three key audiences in mind: the Academy Administrator who needs a daily-use tool, the Computer Science student who can study and extend the codebase, and the frontend developer who can demonstrate professional skills through this portfolio-quality project.

3. Problem Statement

Prior to the development of this system, IQRA Academy faced the following challenges in its administrative processes:

  • Manual Record Keeping: Student enrollment data was maintained in physical registers, leading to data loss, illegibility, and difficulty in retrieval.
  • Inefficient Attendance Tracking: Daily attendance was recorded on paper, making it impossible to calculate overall attendance percentages quickly or identify habitually absent students.
  • Unorganized Fee Management: Fee payment status was tracked informally, often leading to errors in collection and difficulty in generating financial summaries.
  • No Real-time Analytics: Management had no quick overview of key statistics such as enrollment per course, attendance rates, or fee collection totals.
  • Cost and Complexity Barrier: Deploying a full-stack system would require server infrastructure, database management, and ongoing maintenance costs — prohibitive for a small to medium-sized academy.

4. Objectives

The primary objectives of the IQRA Academy Student Management System are:

  1. To develop a fully functional, browser-based Student Management System that requires no backend infrastructure.
  2. To implement secure authentication with session management using browser storage APIs.
  3. To provide a comprehensive student enrollment module with automatic roll number generation and teacher assignment.
  4. To build a date-based attendance management system with percentage calculations and bulk operations.
  5. To create a monthly fee management module with instant toggle updates and financial summaries.
  6. To present real-time data visualizations using Chart.js for informed administrative decision-making.
  7. To deliver a responsive, accessible, and visually professional ERP-style interface.
  8. To demonstrate the full capability of pure frontend technologies in building enterprise-grade applications.

5. Scope of the Project

In Scope: Student CRUD, automated roll numbers, teacher assignment, date-wise attendance with bulk marking, monthly fee tracking, dashboard analytics, dark mode, password management, LocalStorage persistence, responsive UI, toast notifications.
Out of Scope: Backend API, database server, multi-user concurrent access, email notifications, file uploads, PDF export (in v1.0), mobile app version.

The system is intended for single-admin use within a single browser session. Data is stored locally per browser and does not synchronize across devices or users.

8. System Architecture

The system follows a Single-Page Application (SPA) inspired Multi-Page Architecture:

Page Layer

Six HTML pages (index, dashboard, students, attendance, fees, settings) each load their specific JS module after the shared auth.js base module.

Module Layer

  • auth.js — Shared foundation: authentication, layout init, toast system, shared data helpers, course/teacher constants
  • dashboard.js — Statistics computation + Chart.js rendering
  • students.js — Student CRUD, search, auto roll number, modal management
  • attendance.js — Date-based attendance marking, percentage calculation
  • fees.js — Monthly fee status toggle, financial statistics
  • settings.js — Dark mode, password management, data operations

Data Layer

All data is stored in localStorage as JSON-serialized objects. Data is read and written directly within each module using shared helper functions defined in auth.js.

10. User Interface Description

10.1 Login Page (index.html)

The login page features a full-screen gradient background (dark navy to royal blue) with animated floating particles and a glassmorphism login card. The card contains the IQRA Academy logo icon, branding text, username/password fields with icons, a password visibility toggle, and a gradient sign-in button. Demo credentials are displayed for convenience.

10.2 Dashboard (dashboard.html)

The dashboard provides an executive overview with:

  • Statistics Grid: 7 color-coded glassmorphism stat cards (blue, indigo, purple, green, red, teal, orange) with animated number counting on load
  • Charts Row: Students per Course (bar chart), Attendance Overview (doughnut), Fee Status (doughnut) — all responsive and theme-aware
  • Recent Admissions Table: Latest 8 students with avatar initials, roll number badges, and formatted dates

10.3 Student Management (students.html)

A professional data table with all student records. Features a search bar (real-time, searches 4 fields simultaneously), course and gender dropdowns, and result counter. The Add/Edit modal (glass overlay) contains a 9-field form with auto-assignment behaviors. A confirmation dialog with emoji prevents accidental deletions.

10.4 Attendance (attendance.html)

Date picker defaults to today's date. 4 stat cards show real-time counts. Each student row has a segmented control with Present/Absent buttons that highlight green/red when active. Click the same button again to unmark. "Mark All" buttons enable bulk operations. The "Overall %" column shows calculated attendance rates with color coding.

10.5 Fee Management (fees.html)

A month picker (input type="month") controls the billing period. 4 financial stat cards show paid count, unpaid count, total collection, and pending amount. Each table row has a custom checkbox — clicking it instantly toggles paid/unpaid status and saves to LocalStorage without any save button. Paid rows get a green background highlight and a green "Paid" badge.

10.6 Settings (settings.html)

Clean settings layout with card-style sections. The Appearance section has a custom CSS toggle switch for dark mode. The Password section has three fields with individual visibility toggles and client-side validation. The System Information section shows read-only stats. The Danger Zone (red border) has a "Clear All Data" button with a confirmation modal.

10.7 Layout System

All authenticated pages share a consistent layout: a 260px fixed sidebar (dark navy gradient) with brand logo, navigation items, and user info footer; a 64px sticky navbar with sidebar toggle (mobile), breadcrumb, clock, and theme toggle; and a flexible main content area (max-width 1400px, centered). On mobile (≤768px), the sidebar slides in from the left with an overlay backdrop.

14. Conclusion

The IQRA Academy Student Management System successfully achieves all its stated objectives. By leveraging modern browser APIs and pure frontend technologies, the system delivers a professional, enterprise-grade ERP experience without any backend infrastructure. The modular JavaScript architecture ensures maintainability and extensibility, while the responsive CSS design guarantees usability across all device types.

This project demonstrates that fully-featured management systems can be built effectively using only HTML, CSS, and JavaScript — making such applications accessible to institutions that cannot afford complex infrastructure, and proving the depth of capability available to skilled frontend developers.

The system is immediately deployable, requiring nothing more than a modern web browser. It is suitable for real-world use at IQRA Academy and serves as a strong portfolio piece demonstrating advanced frontend development skills.
